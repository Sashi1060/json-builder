Portfolio System Documentation

The complete guide to generating, setting up, and customizing your portfolio.

👋 Welcome!

Thank you for downloading the Portfolio System! If you are reading this, you have successfully extracted the ZIP archive.

Inside this folder, you will find:

  • Templates Folder: Contains beautiful, ready-to-deploy static HTML/CSS portfolio templates.
  • Json_creator Folder: Contains a Python-based generator app to securely compile your portfolio data into a single file.

Follow the steps below to generate your custom data and apply it to the template of your choice!

1. How to Setup & Run the JSON Creator App

The JSON Creator is a Streamlit application used to securely upload images and generate the data.json file required by the templates.

Prerequisites

  • Python 3.8+ must be installed on your system.
  • An active internet connection to download dependencies.

Installation Steps (Using Virtual Environment)

It is highly recommended to use a virtual environment to manage dependencies locally. Follow the steps below based on your Operating System:

  1. Open your terminal / command prompt and navigate to the Json creator folder (where app.py is located).
  2. Create a Virtual Environment:
    • Windows: 
      python -m venv venv
    • macOS / Linux: 
      python3 -m venv venv
  3. Activate the Virtual Environment:
    • Windows (PowerShell - Default): 
      venv\Scripts\Activate.ps1
    • Windows (Command Prompt): 
      venv\Scripts\activate.bat
    • macOS / Linux: 
      source venv/bin/activate
  4. Install Required Libraries: Once the virtual environment is activated, run: 
    pip install streamlit cloudinary imagekitio

Running the App

Make sure your virtual environment is activated, then run the following command in the same directory to start the application:

  • Windows: 
    streamlit run app.py
  • macOS / Linux: 
    python3 -m streamlit run app.py

This will automatically open the generator form in your default web browser (usually at http://localhost:8501).

2. How to Generate your Portfolio JSON

Once the generator is running in your browser, follow these steps to create your data file:

  1. Configure Image Hosting & URLs:
    • Using Image URLs: If your assets (profile picture, project thumbnails) are already present in a cloud service (e.g., Imgur, GitHub, AWS S3, or any public link), you can simply paste the direct URL into the respective fields. You can select "None" for Image Service.
    • Uploading Images: If you want to upload local images directly through the app, choose whether you want to use Cloudinary or ImageKit to host your uploaded images, and enter your respective API keys. The app will upload them to your cloud and automatically generate the URLs for the JSON file.
  2. Fill in the Form: Enter your Name, Headline, About description, Skills, and Contact Email/Location.
  3. Add Projects & Publications: Expand the respective sections to add projects (with Live/GitHub links and thumbnails) and any research publications you have.
  4. Add Social Links: Fill in the URLs for your LinkedIn, GitHub, Twitter, and Personal Website.
  5. Generate & Download: Scroll to the very bottom, view the generated code, and click the Download JSON button. This will download a file named portfolio.json.

3. How to Use & Apply Data to the Templates

Once you have your generated JSON file, applying it to your portfolio template is extremely simple and requires no backend server!

  1. Open the Templates folder and select the template you wish to use.
  2. Open that template's index.html file in any modern web browser.
  3. In the navigation bar of the portfolio, click the { } (Settings) icon.
  4. You will be prompted for an admin PIN. The default PIN is 0000.
  5. Once unlocked, open your downloaded portfolio.json file in a text editor (like Notepad), copy all its contents, and paste it into the secure terminal interface in the browser.
  6. Click Apply ↵. Your portfolio will immediately update with your new information!
Note: The data you paste into the browser is securely saved locally in your browser's localStorage. Anyone who visits your live website will see the data.json file you ultimately deploy alongside your website. To permanently deploy your changes, simply copy your downloaded portfolio.json file into the template folder and rename it to data.json, replacing the default file.

4. How to Change the Settings Password (PIN)

The templates are strictly front-end applications. Because there is no backend server, the PIN is used primarily as a UI barrier to prevent casual browsing or accidental changes to your LocalStorage.

To change the default 0000 PIN to your own secure 4-digit PIN:

  1. Open the folder of the Template you are using.
  2. Open the script.js file in any code editor (like VS Code, Notepad++, etc).
  3. Near the very top of the file (usually around Line 10), locate the following variable: 
    const ADMIN_PIN = '0000';
  4. Change 0000 to your desired 4-digit combination. For example: 
    const ADMIN_PIN = '1234';
  5. Save the script.js file. Your new password is now active!
Note: This portfolio is built for convenience and quick deployment. The downloaded user can modify it as they please.
